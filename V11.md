## 一、整体架构

### 1.1 拆分范围

本次拆分的范围：**Event_Core 独立部署 + SDLC_Jira 独立部署 + SDLC_Redis 独立部署**。SDLC_Event 保持 jar（作为 Event_Core 的内部引擎），SDLC_Admin 和 SDLC_Export 保持 jar 不变，SDLC_POC 保持现状作为临时网关。

### 1.2 拆分原则

**怎么判断一个模块要不要拆成 client/server？**

只有同时满足下面两个条件的，才值得拆：

- 模块内部有"实现代码"被外部通过本地 import 直接引用（不只是 Feign 调用）
- 这些实现代码搬到独立进程后，外部还需要引用它的类型来做本地开发

Event 模块：SDLC_Event 是纯引擎（Validator/Handler 实现），只被 Event_Core 内部调用，外部不需要 import。**所以 SDLC_Event 不需要拆**——它继续做 Event_Core 的内部 jar 依赖即可。

SDLC_Jira 模块：POC 需要 import JiraFeignClient 和 DTO，但不需要 import JiraGateway 的实现。**一个模块就够了**——把 Feign 接口放 api/ 包，实现放 service/ 包，POC 依赖整个 jar 但只调 api/ 的内容。

### 1.3 目标结构

```

│
├── SDLC_Common (jar)              ← 最底层：工具类、基础配置、公共常量、异常定义
│
├── SDLC_Admin (jar)               ← 登录+权限，保持 jar
│   ├── 被 POC、Event_Core、Jira Server 依赖

│   └── 只被 Event_Core 依赖
```

### 1.5 关键依赖规则

- POC 依赖 Event_Core（拿 EventFeignClient + DTO），运行时通过 Feign HTTP 调用
- POC 依赖 SDLC_Jira（拿 JiraFeignClient + DTO），运行时通过 Feign HTTP 调用
- POC 依赖 SDLC_Redis（拿 RedisFeignClient + DTO），运行时通过 Feign HTTP 调用
- Event_Core 依赖 SDLC_Event（jar 引擎，JVM 内调用 Validator/Handler）
- SDLC_Event 不依赖任何业务模块，只依赖 Common
- Common 被所有模块依赖，Common 不依赖任何人

## 二、POM 改造 — 根 POM

### 2.1 根 POM 的定位

根 POM 不做 Spring Boot 的 parent，只做**版本统一管理**。各 bootable 模块（POC、Event_Core、Jira、Redis）各自通过 dependencyManagement import 的方式拿到 spring-boot-starter-parent 的能力。

### 2.2 改造原则：依赖只放该放的地方

| 原来位置 | 动作 | 原因 |
|------|------|------|
| 根 POM `<dependencies>` 里的项目内部模块 | 移到 `<dependencyManagement>`（只声明版本） | 每个模块需要的内部依赖不同 |
| 根 POM `<dependencies>` 里的 spring-boot-starter-web | 移到 `<dependencyManagement>` | 只有 bootable 模块需要 |
| 根 POM `<dependencies>` 里的 mybatis-plus、ojdbc11 | 移到 `<dependencyManagement>` | 只有访问 DB 的模块需要 |
| 根 POM `<dependencies>` 里的 knife4j | 移到 `<dependencyManagement>` | 只有 bootable 模块需要 |
| 根 POM `<dependencies>` 里的 hutool-all | 移到 `<dependencyManagement>` | 留给每个模块自己决定 |
| lombok | 保留在根 POM `<dependencies>` | 所有模块都需要 |
| spring-boot-starter-test | 保留在根 POM `<dependencies>` | 所有模块都需要 |

## 三、Event_Core 独立服务 + SDLC_Event 引擎

### 3.1 改造前

```
SDLC_Event_Core/   → REST Controller + Service（门面，当前被 POC 本地 import 调用）
SDLC_Event/        → Validator/Handler 实现（引擎）
```

前端调 Event_Core 的 Controller → Controller 调 Service → Service 调 SDLC_Event 的 Validator/Handler。

POC 的 Service 内部也可能直接注入 Event_Core 的 Service 来调事件校验。

### 3.2 改造后

**SDLC_Event（jar，不变）**：保持原样，纯引擎，Validator/Handler 实现。

**SDLC_Event_Core（bootable，:8081）**：新增 Feign 接口 + DTO，让 POC 通过 HTTP 调用不再是本地 import。控制器签名和 Feign 接口强绑定。

### 3.3 SDLC_Event_Core 最终包结构

```
SDLC_Event_Core/src/main/java/com/sdlc/event/core/
│
├── EventCoreApplication.java        ← 启动类（原来就有，保留）
│
├── api/                             ← 【新建】对外暴露的 Feign 合同
│   ├── feign/
│   │   └── EventFeignClient.java    ← POC 通过 Feign 调 Event_Core 的合同
│   └── dto/                         ← 原来 Controller 用的请求/响应 DTO
│       ├── ValidationRequest.java
│       ├── ValidationResult.java
│       ├── ExecutionRequest.java
│       └── ExecutionResult.java
│
├── controller/                      ← 原来就有，保留
│   └── EventController.java        ← 原来前端调的，现在 implements EventFeignClient
│
├── service/                         ← 原来就有，保留
│   └── EventExecutionService.java   ← 编排调用 SDLC_Event 的 Validator/Handler
│
└── config/
    ├── SwaggerConfig.java
    └── WebConfig.java
```

**EventFeignClient：**

```java
package com.sdlc.event.core.api.feign;

import com.sdlc.event.core.api.dto.*;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@FeignClient(
    name = "sdlc-event-core",
    url = "${event.core.service.url:http://localhost:8081}"
)
public interface EventFeignClient {

    @PostMapping("/api/event/validate")
    List<ValidationResult> validate(@RequestBody ValidationRequest request);

    @PostMapping("/api/event/execute")
    ExecutionResult execute(@RequestBody ExecutionRequest request);

    @GetMapping("/api/event/health")
    String health();
}
```

**EventController 改造（关键！）：**

```java
@RestController
@RequestMapping("/api/event")
@RequiredArgsConstructor
public class EventController implements EventFeignClient {

    private final EventExecutionService executionService;

    @Override
    @PostMapping("/validate")
    public List<ValidationResult> validate(@RequestBody ValidationRequest request) {
        return executionService.validate(request);
    }

    @Override
    @PostMapping("/execute")
    public ExecutionResult execute(@RequestBody ExecutionRequest request) {
        return executionService.execute(request);
    }

    @Override
    @GetMapping("/health")
    public String health() {
        return "OK";
    }
}
```

Controller 直接 `implements EventFeignClient`——改 Feign 接口时两边一起报编译错，不会出现接口改了 Controller 没改的情况。

### 3.4 POC 调用方式的变化


### 6.3 最容易出错的地方

**坑一：Maven 双 parent 问题。** bootable 模块（Event_Core、Jira、POC、Redis）需要 Boot Parent 的能力来打 fat jar，但 Maven 不支持多继承。解决：每个 bootable 模块的 parent 写根 POM，然后在自己的 `<dependencyManagement>` 中用 `<scope>import</scope>` 引入 spring-boot-starter-parent。

**坑二：MyBatis mapper 路径冲突。** 多服务共用数据库时，每个服务的 `mybatis-plus.mapper-locations` 只扫自己模块的路径。

**坑三：子模块依赖怎么补。** 根 POM 的 `<dependencies>` 被大幅精简后，原来靠继承拿到的依赖会丢失。最佳做法：先不手动补，用 `mvn compile` 编译报错来定位缺什么。报错说缺哪个类，就把对应依赖加到那个模块的 pom.xml。

## 七、RequestContext — 跨服务上下文传递

### 7.1 问题场景

拆成四个独立服务后，一个请求可能经过多个服务。每个服务都需要知道"谁在操作""哪个租户""关联哪个 Jira Instance"。现在这些信息散落在方法参数里，层层传递。

### 7.2 设计目标

任何地方都能无感获取上下文信息，不需要 Controller → Service → DAO 层层传参。

### 7.3 实现机制：Filter + ThreadLocal + Feign RequestInterceptor

**RequestContext（放在 SDLC_Common）：**

```java
package com.sdlc.common.context;

import lombok.Data;
import java.util.Map;

@Data
public class RequestContext {
    private String userId;
    private String username;
    private String accessToken;
    private String instanceId;
    private String cloudId;
    private String siteUrl;
    private String tenantId;
    private String traceId;
    private String requestId;
    private Map<String, String> extras;
}
```

**Current 工具类（放在 SDLC_Common）：**

```java
package com.sdlc.common.context;

public class Current {

    private static final ThreadLocal<RequestContext> CONTEXT = new ThreadLocal<>();

    public static void set(RequestContext context) { CONTEXT.set(context); }
    public static void clear() { CONTEXT.remove(); }

    public static String userId() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getUserId() : null;
    }
    public static String username() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getUsername() : null;
    }
    public static String accessToken() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getAccessToken() : null;
    }
    public static String instanceId() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getInstanceId() : null;
    }
    public static String tenantId() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getTenantId() : null;
    }
    public static String traceId() {
        RequestContext ctx = CONTEXT.get();
        return ctx != null ? ctx.getTraceId() : null;
    }
}
```

**ContextFilter（每个服务各自配置）：**

```java
@Component
@Order(1)
public class ContextFilter implements Filter {

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest httpRequest = (HttpServletRequest) request;

        RequestContext context = new RequestContext();
        context.setUserId(httpRequest.getHeader("X-User-Id"));
        context.setUsername(httpRequest.getHeader("X-Username"));
        context.setAccessToken(httpRequest.getHeader("Authorization"));
        context.setTenantId(httpRequest.getHeader("X-Tenant-Id"));
        context.setInstanceId(httpRequest.getHeader("X-Instance-Id"));

        String traceId = httpRequest.getHeader("X-Trace-Id");
        context.setTraceId(traceId != null ? traceId : UUID.randomUUID().toString());

        try {
            Current.set(context);
            chain.doFilter(request, response);
        } finally {
            Current.clear();
        }
    }
}
```

**ContextFeignInterceptor（放在 SDLC_Common）：**

```java
@Configuration
public class ContextFeignInterceptor implements RequestInterceptor {

    @Override
    public void apply(RequestTemplate template) {
        addIfNotNull(template, "X-User-Id", Current.userId());
        addIfNotNull(template, "X-Username", Current.username());
        addIfNotNull(template, "Authorization", Current.accessToken());
        addIfNotNull(template, "X-Tenant-Id", Current.tenantId());
        addIfNotNull(template, "X-Instance-Id", Current.instanceId());
        addIfNotNull(template, "X-Trace-Id", Current.traceId());
        addIfNotNull(template, "X-Request-Id", Current.requestId());
    }

    private void addIfNotNull(RequestTemplate template, String name, String value) {
        if (value != null) template.header(name, value);
    }
}
```
